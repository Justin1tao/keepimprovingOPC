### 🤖 项目一：Winky-bot（重点）

**一句话定位**：
> 基于单显示器视觉跟随的智能光标引导桌面机器人系统
> 让 AI 看懂屏幕，用机器人引导用户完成桌面任务

**项目简介**：
```
Winky-bot 是一个具身 AI 桌面助手系统，解决"自然语言意图到
桌面空间坐标的最后一公里"问题。

当用户说"帮我打开屏幕录制权限"时，系统会：
1. 🖥️ 看屏幕 → 截取当前桌面画面
2. 🧠 理解任务 → AI 识别目标 UI 元素位置
3. 🎯 指向目标 → 辅助光标 overlay 精准引导
4. 🤖 机器人反馈 → 桌面机器人转头、眼睛看向目标，语音提示

核心价值：让不熟悉电脑操作的人（老人、儿童、软件初学者）
也能轻松完成复杂桌面任务。
```

**项目背景**：
- ✅ 研究生电子设计大赛参赛项目
- ✅ 校级创业项目，已完成 Demo 演示与成品开发
- ✅ 持续迭代中：正在开发多步骤 AI 提示与连续交互功能

**技术架构**（点击展开）：
```
┌─────────────────────────────────────────────┐
│  ☁️ 云端 AI（多模态视觉 API）                 │
│  Gemini / MiMo / OpenAI-compatible          │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  💻 Mac 端（Clicky 服务）                     │
│  屏幕截图 → 视觉定位 → 坐标映射 → Overlay引导 │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  🔗 Harness 微内核（状态机 + EventBus）       │
│  三段式调度 + ACK + 异常恢复 + trace_id       │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  🤖 具身执行（ESP32 机器人）                  │
│  头部转向 + 眼睛朝向 + 语音提示               │
└─────────────────────────────────────────────┘
```

**核心亮点**（点击展开）：
- **具身 UI Grounding** - 屏幕定位 ↔ 机器人动作实时同步
- **三段式 Agent 架构** - Mac 感知 + Harness 调度 + ESP32 执行 解耦
- **隐私优先设计** - 截图内存态处理，零落盘
- **多步骤任务引导** - 支持连续交互，分步骤引导（开发中）

**项目初心**：
> "让人类在 AI 时代依然能独立掌控工具"
> 我们 00 后已经不会手写字了，未来的孩子可能连"找到按钮"都不会。
> Winky-bot 不是让 AI 替人做事，而是让人借助 AI 更好地使用工具。

### 🎓 项目二：考研调剂 AI 系统（重点）

**一句话定位**：
> 基于 LangGraph 的考研调剂 AI 增效工作系统
> AI 辅助采集 + 人工决策，提升咨询效率 3-5 倍

**项目简介**：
```
为提升个人考研咨询服务效率而自研的 AI Agent 工作系统。
通过 LangGraph 构建半自动化工作流，实现调剂信息采集、
智能分析、记忆存储、动态优化等模块的协同工作。

设计哲学：AI 辅助采集 + 人工决策
AI 负责信息聚合与初步筛选，人类（资深顾问）进行官方源核实
与最终决策，既保证效率，又确保信息的准确性和权威性。
```

**个人角色**：
- 系统架构师 + 考研调剂顾问（亲历者视角，2025届考生）

**技术架构**（点击展开）：
```
- 后端框架：LangGraph + FastAPI
- AI 引擎：豆包大模型 + 联网搜索 API
- 前端界面：本地 Web 界面（HTML/CSS/JS）
- 存储方案：Markdown 记忆库 + 本地文件管理
```

**核心模块**（点击展开）：

**1. 智能信息采集模块**
- 自动抓取目标院校调剂信息（院校官网、研招网等）
- 基于用户输入的专业代码，每日追踪最新调剂动态
- 工作模式：AI 辅助采集 + 人工核实（确保信息官方性）

**2. 数据分析与决策辅助模块**
- 历年分数段去向分析：输入分数+专业，聚合同分数段录取院校统计
- 院校层次筛选：按 985/211/一本/二本分类
- 考生背景分析：结合本科层次、一志愿、项目经历等多维度评估
- 专业代码追踪：每日自动查询目标专业的最新调剂新闻与动态

**3. Harness 工程与工作流自动化**
- 固定化的提示词模板库（Harness）
- 一键信息提取：输入基础信息，自动解析并填充到工作流
- 流程优化：单次咨询响应时间从 30 分钟缩短至 5 分钟

**4. 分层服务与记忆系统**
- **一对一全程服务**：专属 Markdown 记忆库，持续更新，支持多轮对话
- **批量咨询服务**：Harness 标准化输出，响应快速

**5. 动态学习与进化机制**
- 经验反馈循环：用户输入"更新到总要求 md"，系统自动整合新规则
- Prompt 动态优化：基于对话反馈，持续迭代工作流配置
- 个性化适配：随使用时间增长，系统越来越贴合个人习惯

## 12. 交互设计规范

### ✨ 动画效果

**鼠标光圈**：
- 颜色：蓝白色渐变（#00f5ff → #ffffff）
- 大小：半径 150px
- 模糊：高斯模糊 50px
- 透明度：0.3
- 跟随：平滑跟随，延迟 0.1s

**动态背景**：
- 粒子系统：tsparticles
- 粒子数量：50-100 个
- 粒子颜色：#00f5ff（霓虹青）
- 连线：显示粒子间的连线
- 网格线：微妙的数字网格，低透明度

**滚动动画**：
- 模块淡入：opacity 0 → 1，向上浮动 50px
- 时间轴：节点依次点亮，线条扫描
- 卡片：hover 时霓虹边框 + 3D tilt（最大 5°）

**打字机效果**：
- Hero 副标题逐字显示
- 打字速度：50ms/字
- 光标闪烁：1s 周期

### 📱 响应式设计

**桌面端（>1024px）**：
- 完整视觉效果
- 动态背景 + 鼠标光圈
- 卡片 3D tilt 效果

**平板端（768-1024px）**：
- 保留核心动画
- 减少粒子数量
- 简化 3D 效果

**移动端（<768px）**：
- 禁用动态背景（省电）
- 禁用鼠标光圈
- 单列布局
- 简化动画，保证流畅
