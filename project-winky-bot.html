Winky · Multi-Agent Cursor Companion

面向未来人机协同的智能光标跟随桌面机器人。

Winky 是一款多 Agent 智能光标跟随桌面机器人。它关注的不是“替用户点完所有按钮”，而是让 AI 理解用户目标、看懂当前屏幕，并通过光标高亮、语音、表情与眨眼等具身反馈，引导用户完成复杂软件里的下一步操作。

Problem Framing

软件越来越复杂，真正的门槛是找不到正确入口。

数字时代的很多用户并不是不会使用电脑，而是看不懂路径、找不到入口、无法判断下一步该点哪里。Winky 把这种操作断点转化为可视化、可听见、可追踪的协同引导。

任务理解

用户只需要说出目标，例如“帮我找到文件重命名的位置”，系统先把自然语言目标拆解成可执行的操作意图。

视觉定位

视觉 Agent 理解当前屏幕，识别按钮、菜单、输入框等 UI 元素，把抽象意图映射到具体目标区域。

具身引导

光标高亮、机器人语音、表情、眨眼和视线方向共同给出反馈，让用户知道下一步在哪里、为什么这么做。

Project Flow

从用户目标，到 Agent 协同，再到可感知的桌面引导。

01

用户提出目标

用户不需要描述界面结构，只需要说出想完成的任务，例如找入口、改设置、完成某个办公软件操作。

02

意图 Agent 分析需求

系统判断用户真正要完成的操作，并把目标整理成可追踪的引导链路，而不是直接给一段笼统教程。

03

视觉 Agent 定位目标

多模态视觉模型理解当前屏幕，识别目标 UI 元素的位置，将操作意图落到具体坐标和界面区域。

04

光标与机器人共同引导

辅助光标高亮目标，桌面机器人通过语音、表情、眨眼和视线方向给出具身反馈，引导用户完成下一步。

Architecture

多 Agent 协同，把复杂界面变成可跟随的操作路径。

Winky 的核心价值在于把传统“人理解界面、寻找操作”转变为“AI 理解任务、引导人完成操作”。适用于办公软件辅助、教育学习指导、老年人与新手数字化帮助等场景。

视觉 Agent：看懂当前屏幕

多模态视觉 API 负责理解屏幕内容，识别目标按钮、菜单、文件、输入框等 UI 元素，并输出可用于光标引导的空间位置。
